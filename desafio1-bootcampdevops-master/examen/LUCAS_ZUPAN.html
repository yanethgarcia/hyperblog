Examen Git y Github

Alumno: Lucas Zupan

  1. Que es un sistema de control de versiones?

    Los sistemas de control de versiones son un tipo de aplicación con la función de gestionar los cambios de un conjunto de elementos. Deben proporcionar los mecanismos para almacenar los archivos, reliazar cambios sobre estos y registrar historicamente todas las modificaciones.

  2. Cual es la diferencia entre git y github?

    Git es el software de control de versiones, mientras que GitHub es una plataforma donde alojar repositorios de git. Resuelve muchas cosas como gestionar los permisos a otros usuarios para que colaboren en un mismo repositorio, entre otras funciones.

  3. Cómo creo una rama en git?

    Agregar una nueva rama localmente: git branch <nueva_rama>, o con el comando git checkout -b <nueva_rama> para crear y cambiar de rama simultaneamente.
    Agregar una nueva rama al repositorio remoto: git push --set-upstream origin <nueva_rama>

  4. Cómo clono un repositorio?

    Clono un repositorio con el comando: git clone <ruta_repositorio>
    ruta_repositorio: Puede ser una URL para descargar por HTTPS, o un host remoto para descargarlo por SSH (usuario@host:ruta)

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    Con el comando git pull

  6. Cómo puedo gener un tag?

    Genero un tag con el comando git tag -a <nombre_tag>
    Lo subo al repositorio con git push origin --tags

  7. Cómo hago una mezcla de ramas?

    Primero me posiciono en la rama de destino, por ejemplo main git checkout main
    Luego mezclo con la otra rama git merge <otra_rama>

  8. Cómo puedo moverme a un commit especifico?

    Debemos conseguir el hash del commit con git log, y luego un git checkout <hashDelCommit>

  9. Cómo puedo borrar una rama?

    Borrar una rama localmente: git branch -d <rama>
    Borrar una rama del repositorio remoto: git push origin --delete <rama>

  10. Cómo puedo borrar un tag?

    Borrar un tag localmente: git tag -d <tag>
    Borrar un tag del repositorio remoto: git push origin --delete origin <tag>

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con el comando git status

  12. Para que sirve el comando git remote -v?

    Para ver con que repositorio remoto esta asociado tu repositorio local

  13. Cómo le digo a git cual es mi email y correo?

    git config user.email "<correo>"

  14. Cuales son los pasos para conectar git con github?

    En el repo local ejecutar: git remote add origin <ruta_repositorio>

  15. Cómo puedo crear un alias en git?

    Creo un alias con el comando: git config --global alias.<nombre_alias> <comando>
    Donde nombre_alias es la nueva forma con la que se podrá llamar al comando.
    Por ejemplo: git config --global alias.st status
    Ahora se puede ejecutar git st con el mismo resultado que git status
