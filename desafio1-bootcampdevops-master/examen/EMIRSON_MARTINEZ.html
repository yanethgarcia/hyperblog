Examen Git y Github

Alumn@: Emirson Martinez

  1. Que es un sistema de control de versiones?

    Es un sistema que registra los cambios realizados en un archivo o conjunto de archivos a lo largo del tiempo, de modo que se pueda recuperarversiones especificas de los cambios realizados, es especialmente util en el mundo DEVOPS, donde grupos de personas tienen acceso al mismo codigo fuente y realizan modificaciones sobre este.

  2. Cual es la diferencia entre git y github?

    Git es una herramienta de control de versiones, Github es un portal que ofrece el servicio de almacenamiento de repositorios en la nube permitiendo a personas o equipos de trabajo colaborar en un mismo codigo fuente.

  3. Cómo creo una rama en git?

    git checkout -b nombre_rama

  4. Cómo clono un repositorio?

    git clone URL_repositorio

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    git pull origin master

  6. Cómo puedo gener un tag?

    git tag tag_name

  7. Cómo hago una mezcla de ramas?

    git merge branch_name

  8. Cómo puedo moverme a un commit especifico?

    git reset HEAD~[n]

  9. Cómo puedo borrar una rama?

    git branch -d branch_name

  10. Cómo puedo borrar un tag?

    git tag -d tag_name

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    git diff

  12. Para que sirve el comando git remote -v?

    muestra las conexiones remotas con otros repositorios.

  13. Cómo le digo a git cual es mi email y correo?

    git config --global user.email user@email.com

  14. Cuales son los pasos para conectar git con github?

    Configurar usuario y configurar correo en GIT

  15. Cómo puedo crear un alias en git?

    git config --global alias.[alias_name] command_name
