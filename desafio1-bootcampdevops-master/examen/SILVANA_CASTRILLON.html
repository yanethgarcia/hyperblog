Examen Git y Github

Alumn@: [Nombre Completo]

  1. Que es un sistema de control de versiones?

    [Un Sistema de Control de versiones permite a los equipos de desarrollo realizar diferentes cambios/tareas/funcionalidades del software de manera distribuida permitiendo trabajar de manera local a cada desarrollar su tarea para despues unificarlo en un repositorio central y pasar a realizar pruebas sobre ese nuevo codigo]

  2. Cual es la diferencia entre git y github?

    [Git es el control de versiones y GitHub una herramienta que permite hacer esto]

  3. Cómo creo una rama en git?

    [git branch NOMRE_DE_LA_RAMA de estructura Type/Name]

  4. Cómo clono un repositorio?

    [Localemente con Git Clone URL del repositorio, en GitHub con Fork]

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    [con el comando Git pull permite actualizar la rama local en la que se esta trabajando con todos los commits de la rama remota de GitHub]

  6. Cómo puedo gener un tag?

    [GitHub lo permite desde la interfaz grafica en la creacion del tag, creo que el comando es git tag ...]

  7. Cómo hago una mezcla de ramas?

    [git merge combina la rama remota con la local]

  8. Cómo puedo moverme a un commit especifico?

    [Git show muestra todos los commits]]

  9. Cómo puedo borrar una rama?

    [git branch -d nombre de la rama]

  10. Cómo puedo borrar un tag?

    [Desde la interfaz visual y con git tag -d]

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    [git diff]

  12. Para que sirve el comando git remote -v?

    [se utiliza para especificar los extremos remotos sobre los que operarán los comandos de sincronización.]

  13. Cómo le digo a git cual es mi email y correo?

    [con los comandos git config --global user.name y git config --global user.email]

  14. Cuales son los pasos para conectar git con github?

    [1. Descargar Git en tu pc; 2. Configuraciones globales como el nombre, correo; 3. autenticar github desde git]

  15. Cómo puedo crear un alias en git?

    [git config --global alias.[alias]]
