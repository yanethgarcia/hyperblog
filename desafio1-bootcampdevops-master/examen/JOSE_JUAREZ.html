Examen Git y Github

Alumn@: [Jose Juarez]

  1. Que es un sistema de control de versiones?

    Es una herramienta que nos permite gestionar los cambios en nuestros archivos de codigo, para controlar y mejorar el flujo de trabajo entre los equipos de desarrollo

  2. Cual es la diferencia entre git y github?

    Git es la herramienta de control de versiones, Github es un servicio que permite almacenar y gestionar repositorios git

  3. Cómo creo una rama en git?

    Una rama en git se crea con el siguiente comando:

    git branch [nombre_rama]

  4. Cómo clono un repositorio?

    Un repositorio en git se clona con el siguiente comando:

    git clone [direccion_repositorio_a_clonar]

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    Tomar los ultimos cambios del repositorio se hace con este comando:

    git pull

  6. Cómo puedo gener un tag?

    Se puede poner un tag con este comando:

    git tag [nombre_tag]

  7. Cómo hago una mezcla de ramas?

    Combinar ramas se hace con este comando:

    git merge [nombre_rama_a_combinar]

  8. Cómo puedo moverme a un commit especifico?

    Se puede seleccionar un commit en particular con este comando:

    git checkout [id_del_commit]

  9. Cómo puedo borrar una rama?

    Se puede borrar una rama con este comando:

    git branch -d [nombre_rama]

  10. Cómo puedo borrar un tag?

    Se puede borrar un tag con este comando:

    git tag -d [nombre_tag]

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Los cambios que estan por enviarse al repositorio se visualizan con este comando:

    git status

  12. Para que sirve el comando git remote -v?

    Este comando permite visualizar la url de los repositorios remotos

  13. Cómo le digo a git cual es mi email y correo?

    Se puede establecer el email y nombre del usuario con este comando:

    git config --global user.email [email_usuario] git config --global user.name "[nombre_usuario]"

  14. Cuales son los pasos para conectar git con github?

    Asumiendo que tenemos un repositorio local previamente configurado y contamos con una cuenta en Github, podemos hospedar nuestros cambios a Github con los siguientes pasos:

    Crear un repositorio en Github y copiar la url del repositorio recien creado

    Ejecutar el comando git remote add origin [url_repositorio]

    Ejecutar el comando git push origin main para reflajar los cambios del repositorio local en el repositorio remoto de GitHub

  15. Cómo puedo crear un alias en git?

    Un alias en git sirve para renombrar comandos a versiones mas cortas, para evitar escribir el comando original completo y se puede hacer con el comando git config --global alias.[nombre_alias] [comando_original] por ejemplo: git config --global alias.co checkout
