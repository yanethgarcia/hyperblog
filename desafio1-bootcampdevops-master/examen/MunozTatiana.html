Examen Git y Github

Alumn@: Tatiana Muñoz

  1. Que es un sistema de control de versiones?

    Es una herramienta que permite gestioanr cambios, es decir, visualizar cambios en equipos, volver atras si se comete un error, comparar versiones, etc del codigo en el que un equipo trabaja

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones, Github es el almacenamiento de repositorios, osea donde se encuentran los codigos en los que se haran cambios con el sistema de control de versiones (GIT)

  3. Cómo creo una rama en git?

    git checkout -b [nombreRama]

  4. Cómo clono un repositorio?

    git clone [HTTPS o SSH]

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    git pull origin [rama]

  6. Cómo puedo gener un tag?

    git tag [nombreTag]

  7. Cómo hago una mezcla de ramas?

    git merge [nombredeRamaAfusionar]

  8. Cómo puedo moverme a un commit especifico?

    Debemos conseguir el hash del commit con Git log, y luego un git checkout [hashDelCommit]

  9. Cómo puedo borrar una rama?

    git branch -d [nombreDelBranch] (Esto es para ramas locales) git push origin --delete [nombreDelBranch] (Para ramas remotas)

  10. Cómo puedo borrar un tag?

    git tag -d [nombreTag] git push origin --delete origin [nombreTag]

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    git status, te dira que archivos han sido creados, modificados o eliminados, los verdes estan listos para enviarse, los rojos no estan agregados en el commit

  12. Para que sirve el comando git remote -v?

    Para ver con que repositorio remoto esta asociado tu repositorio local

  13. Cómo le digo a git cual es mi email y correo?

    git config, podes indicar mail y usuario git config user.email [mail]

  14. Cuales son los pasos para conectar git con github?

    git remote add origin [url]

  15. Cómo puedo crear un alias en git?

    git config --global alias.co [alias]
