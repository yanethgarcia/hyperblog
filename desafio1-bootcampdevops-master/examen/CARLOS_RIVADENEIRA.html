Examen Git y Github

Alumn@: [Carlos Rivadeneira]

  1. Que es un sistema de control de versiones?

    [Es un sistema que permite registrar cambios de archivos en texto plano, asi como 'snapshots' en el tiempo y poder regresar en un tiempo en el pasado si se requiere]

  2. Cual es la diferencia entre git y github?

    [La diferencia es que git es el sistema de control de versiones, mientras que github es la plataforma en la nube donde se alojan los repositorios remotos que se pueden crear y actualizar con git]

  3. Cómo creo una rama en git?

    [Hay varias formas, la más sencilla desde la rama actual es con 'git checkout -b nombre_rama']

  4. Cómo clono un repositorio?

    [Fácil, desde la consola digitamos 'git clone origen_del_repositorio_a_clonar']

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    [La forma más común es con 'git pull' y para ser especifico a bajar cambios de una rama 'git pull origin nombre_rama']

  6. Cómo puedo gener un tag?

    [Esta me tocó googlearla, según mi research es con el comando 'git tag v1.x.x -m "mensaje_del_tag"']

  7. Cómo hago una mezcla de ramas?

    [Por ejemplo desde la rama en la que estamos actualmente, sería así 'git merge rama_con_cambios_a_mezclar']

  8. Cómo puedo moverme a un commit especifico?

    [Para eso necesitamos tener a la mano el hash del commit (para ver dicho hash necesitamos ver el listado de commits con 'git log'), realmente basta con una parte de ese hash para que git se fije que se trata de dicho commit, lo hacemos con 'git checkout hash_string']

  9. Cómo puedo borrar una rama?

    [Con 'git branch -d nombre_rama' podemos hacerlo]

  10. Cómo puedo borrar un tag?

    [Para borrar un tag que no ha sido publicado basta con 'git tag -d nombre_tag', pero si por el contrario ya esta arriba dicho tag entonces seria de la siguiente manera 'git push origin --delete nombre_tag', esta tampoco sabía toco hacer research...]

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    [Con 'git diff' puedo ver los cambios que se han modificado sin agregrarlos al 'stage', por el contrario si ya los agregue al 'stage' se debe adicionar el parametro cached así 'git diff --cached', pero para ver los cambios que están por enviarse basta con un 'git diff origin', con este último podemos ver la diferencia entre lo que está actualmente en el repositorio central vs. nuestros cambios locales]

  12. Para que sirve el comando git remote -v?

    [Sirve para ver la url remota del origin del repositorio central, por ejemplo: la url de github.com donde esta alojado el repositorio tanto para jalar cambios (fetch), como para subirlos (push)]

  13. Cómo le digo a git cual es mi email y correo?

    [Siento que quisiste decir nombre y correo, esta configuración es obligatoria despues de instalar git jaja, serian dos comandos a correr para que la configuración se replique de manera global, 'git config --global user.name "Mi nombre"' y 'git config --global user.email micorreo@correo.com']

  14. Cuales son los pasos para conectar git con github?

    [En el caso de que el repo no cuente con un remote se lo puede definir con el siguiente comando 'git remote add origin https://github.com/user/repo.git']

  15. Cómo puedo crear un alias en git?

    [Desde la configuración global con el comando alias seguido por un . y el nombre del alias, por ejemplo para definir un alias simplificado del checkout 'git config --global alias.co checkout']
